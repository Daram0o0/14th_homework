핵심요구사항) 현재의 모든 구조를 그대로 유지하고, 하드코딩된 Mock데이터를 제거하고, 실제 데이터를 바인딩할 것. 적용 결과를 체크리스트로 반환할 것.
            1) 라이브러리조건
                - API: 이미 설치되어있는 apollo-client를 사용할 것.
                - 이미 설치되어 있는 codegen을 사용할 것.
                - commons/queries/products/하위에 graphql폴더를 만들어 사용되는 쿼리 파일을 만든 후 npm run codegen을 사용해 해당 API를 사용할 것. (구현되어 있음)

            2) 바인딩 API를 적용할 것.
                1. fetchTravelproductsOfTheBest API
                    1-1) API명: fetchTravelproductsOfTheBest
                    1-2) 요청데이터: 없음
                    1-3) 응답데이터: { [Travelproduct!]! }
                    1-4) 이미 queries.ts에 만들어져 있는 요청 데이터에 대해서 해당하는 칸에 바인딩을 시도할 것.

            3) list 변동 값에 따라 리패치할 것.
                3-1) list 컴포넌트의 변동 값(pickedCount)이 변경될 때 carousel도 자동으로 리패치되도록 구현할 것.

            4) 반대로 carousel 변동 값에 따라 list를 리패치할 것.
                4-1) carousel 컴포넌트의 변동 값(fetchTravelproductsOfTheBest의 pickedCount)이 변경될 때 list가 자동으로 리패치되도록 구현할 것.

            5) 로그인된 사용자의 pick 상태 확인 및 초기화
                5-1) /products 페이지에 진입했을 때, 로그인된 사용자가 이미 pick한 상품 목록을 fetchTravelproductsIPicked API를 사용하여 가져올 것.
                5-2) 가져온 상품 목록을 기반으로 pickedProductIds를 초기화하여, 이미 pick한 상품은 채워진 아이콘으로 표시할 것.
                5-3) 로그인되지 않은 사용자의 경우 빈 상태로 초기화할 것.
