[1차 핵심요구사항]

핵심요구사항) 현재의 모든 구조를 그대로 유지하고, 하드코딩된 Mock데이터를 제거하고, 실제 데이터를 바인딩할 것. 적용 결과를 체크리스트로 반환할 것.
            1) 라이브러리조건
                - API: 이미 설치되어있는 apollo-client를 사용할 것.
                - 이미 설치되어 있는 codegen을 사용할 것.
                - commons/queries/products/queries.ts에 사용되는 쿼리 파일을 만든 후 npm run codegen을 사용해 해당 API를 사용할 것.

            2) 바인딩 API를 적용할 것.
                1. toggleTravelproductPick API
                    1-1) API명: toggleTravelproductPick
                    1-2) 요청데이터: 
                    {
                        travelproductId: ID!
                    }
                    1-3) 응답데이터: { Int! }

            3) toggleTravelproductPick 로직
                - 사용자는 한 번만 토글할 수 있을 것.
                - 토글 즉시 fetch되어서 최신화될 것.
                - 사용자가 토글시, outline이 아닌 가득차 있는 아이콘으로 바뀔 것.(눌러진 상태)
                - 로그인된 사용자가 눌러지지 않은 상태에서는 outline일 것.
                - 눌러진 정보는 Apollo Client 캐시와 Context를 사용하여 관리할 것.
                - toggleTravelproductPick 아이콘 클릭 시, 페이지 이동이 되지 않을 것.

            4) 로그인된 사용자의 pick 상태 확인 및 초기화
                4-1) /products 페이지에 진입했을 때, 로그인된 사용자가 이미 pick한 상품 목록을 fetchTravelproductsIPicked API를 사용하여 가져올 것.
                4-2) 가져온 상품 목록을 기반으로 pickedProductIds를 초기화하여, 이미 pick한 상품은 채워진 아이콘으로 표시할 것.
                4-3) 로그인되지 않은 사용자의 경우 빈 상태로 초기화할 것.
                4-4) products가 로드될 때마다 현재 화면에 표시된 상품 중에서 사용자가 pick한 상품만 필터링하여 상태를 업데이트할 것.
