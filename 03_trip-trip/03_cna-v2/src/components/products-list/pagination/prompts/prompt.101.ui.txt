[1차 핵심요구사항]

1. 컴포넌트 구성
  - pagination 컴포넌트는 리스트 하단 중앙에 위치할 것
  - 좌우 화살표 버튼과 페이지 번호 버튼으로 구성
  - 페이지 그룹은 5개씩 표시
  - 현재 페이지는 활성화 스타일 적용

2. 스타일 요구사항
  - 중앙 정렬 (justify-content: center)
  - 화살표 아이콘: 24px 크기, 색상 #333333
  - 비활성화된 화살표: 색상 #c7c7c7, cursor: default
  - 페이지 번호 버튼: 32px x 32px, 색상 #777
  - 활성화된 페이지: 배경색 #f2f2f2, 색상 #000, font-weight: 500
  - 페이지 번호 간격: 16px
  - 화살표와 페이지 번호 간격: 8px

3. 기능 요구사항
  - 왼쪽 화살표: 현재 페이지에서 이전 페이지로 이동 (한 페이지씩)
  - 오른쪽 화살표: 현재 페이지에서 다음 페이지로 이동 (한 페이지씩)
  - 특정 페이지 번호 클릭 시 해당 페이지로 이동
  - 첫 페이지(1페이지)에서는 왼쪽 화살표 비활성화 (클릭 이벤트 차단)
  - 마지막 페이지에서는 오른쪽 화살표 비활성화 (클릭 이벤트 차단)
  - 페이지 변경 시 refetch를 통해 데이터 갱신
  - 페이지 그룹은 5개씩 표시하되, 현재 페이지에 따라 자동 조정

4. 빈 상태 처리 요구사항
  - 상품이 없는 경우 cardGrid에 min-height: 400px 적용
  - "상품이 없습니다." 문구는 중앙 배치 (emptyState 클래스 사용)
  - 빈 상태 스타일: 18px 폰트, #777777 색상, 중앙 정렬

5. 그리드 레이아웃 요구사항
  - 카드 그리드: 5열 2행 구조 (grid-template-columns: repeat(5, 1fr))
  - 카드 간격: 24px (gap: 24px)
  - 카드 이미지 크기: 240px x 200px
  - 카드 최소 높이: 320px
